# 待办与技术债（长期跟踪）

更新时间：2026-02-27

## P0 - 管理面公网暴露收敛（未授权扫描噪声）

- 主题：管理服务器 `controller` 接口存在较高未授权访问扫描量（`auth.unauthorized` 24h 可达数千）。
- 当前状态：
  - 已有 `AUTH_TOKEN` 校验、轻量限流、审计记录与 bot 概览展示。
  - 但管理面仍可能通过公网端口被持续探测，存在噪声与潜在攻击面。
- 优先措施（按顺序）：
  1. 更换强随机 `AUTH_TOKEN`（禁止弱口令/默认值）。
  2. 收敛入口：优先仅开放 `443`（Caddy 反代），关闭公网 `8080`。
  3. 若暂不能关闭 `8080`：UFW 仅放行“节点 IP + 管理员固定 IP”。
  4. 节点侧 `controller_url` 统一切换为 `https://<panel-domain>`。
  5. 可选：接入 fail2ban 针对高频 401/403 自动封禁。
- 验收标准：
  - `admin/overview.security_events.unauthorized_24h` 显著下降。
  - 管理端功能（bot/controller/agent）保持可用，无回归。

## P0 - 协议限速能力补全

- 主题：`VLESS+REALITY` 按用户限速（当前未解决）
- 当前状态：
  - `TUIC` 可通过 B 模式（每用户独立端口）实现精准限速。
  - `VLESS+REALITY` 在当前共用 `443` 架构下，尚未有同等精度的按用户限速方案。
- 影响：
  - 面板“修改限速”目前对 VLESS 用户侧感知不一致，可能出现“参数已更新但数据面未精准生效”。
- 后续候选方案（待评估）：
  1. 节点级/入口级总限速（简单但非按用户）
  2. 按来源 IP 近似限速（精度有限）
  3. VLESS 独立端口/独立入口模型（可精确，但改变当前统一 443 体验）
- 结论：
  - 该问题列为长期未解决项，项目迭代中需持续跟踪，直至有可落地方案。
